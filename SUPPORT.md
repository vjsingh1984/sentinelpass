# Support

## Getting Help

| Need | Best path |
| --- | --- |
| Usage or setup questions | GitHub Discussions / Issues |
| Bug report | Open a GitHub issue with logs and steps |
| Security vulnerability | Follow `SECURITY.md` private reporting |

## Before Opening an Issue

- Confirm latest `develop` or latest release tag behavior.
- Include platform, version, and exact command/UI flow.
- Attach sanitized logs (never include plaintext secrets).

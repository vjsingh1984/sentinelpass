# Mobile Bridge Usage Guide

This guide explains how to use the `sentinelpass-mobile-bridge` library from iOS (Swift) and Android (Kotlin) applications.

## Overview

The mobile bridge provides a safe interface to the SentinelPass core Rust library through platform-native FFI/JNI boundaries.

- **iOS**: Uses C ABI via `sentinelpass_bridge.h` header
- **Android**: Uses JNI bindings via native methods
- **Shared**: All bridge functionality in `bridge.rs`

## iOS Integration (Swift)

### 1. Add the Rust Static Library

Add `sentinelpass-mobile-bridge` as a dependency to your Xcode project:

```swift
// In your Xcode project settings:
// Build Phases → Link Binary With Libraries → Add → sentinelpass-mobile-bridge.a
// Build Settings → Header Search Paths → add path to include/
```

### 2. Import the C Header

```swift
import Foundation
// The header is generated by cbindgen
// Make sure sentinelpass_bridge.h is in your project
```

### 3. Swift Code Example

```swift
class SentinelPassBridge {
    private var vaultHandle: UInt64 = 0

    /// Initialize or unlock the vault
    func unlockVault(vaultPath: String, masterPassword: String) -> Bool {
        vaultPath = vaultPath.cString(using: .utf8)!
        let password = masterPassword.cString(using: .utf8)!

        let errorCode = sp_vault_init(vaultPath, password, &vaultHandle)

        return errorCode == SPErrorCode_Success
    }

    /// Add a password entry
    func addEntry(title: String, username: String, password: String,
                  url: String, notes: String) -> String? {
        let titleC = title.cString(using: .utf8)!
        let usernameC = username.cString(using: .utf8)!
        let passwordC = password.cString(using: .utf8)!
        let urlC = url.cString(using: .utf8)!
        let notesC = notes.cString(using: .utf8)!

        var entryIdPointer: UnsafeMutablePointer<CChar>? = nil

        let result = sp_entry_add(
            vaultHandle,
            titleC,
            usernameC,
            passwordC,
            urlC,
            notesC,
            &entryIdPointer
        )

        guard result == SPErrorCode_Success else {
            return nil
        }

        let entryId = String(cString: entryIdPointer!)
        sp_string_free(entryIdPointer)

        return entryId
    }

    /// List all entries
    func listEntries() -> [EntrySummary] {
        var entriesPointer: UnsafeMutablePointer<UnsafeMutablePointer<SPEntrySummary>?>?
        var count: usize = 0

        let result = sp_entry_list_all(vaultHandle, &entriesPointer, &count)

        guard result == SPErrorCode_Success, let entries = entriesPointer else {
            return []
        }

        var summaries: [EntrySummary] = []

        for i in 0..<count {
            let entry = entries![i].pointee
            let summary = EntrySummary(
                id: String(cString: entry.id!),
                title: String(cString: entry.title!),
                username: String(cString: entry.username!),
                favorite: entry.favorite
            )
            summaries.append(summary)

            // Free strings
            sp_string_free(entry.id!)
            sp_string_free(entry.title!)
            sp_string_free(entry.username!)
        }

        // Free array
        sp_bytes_free(entriesPointer, count)
        return summaries
    }

    /// Generate TOTP code
    func generateTotp(entryId: String) -> TotpCode? {
        let entryIdC = entryId.cString(using: .utf8)!

        var totpCode: SPTotpCode = SPTotpCode()

        let result = sp_totp_generate_code(vaultHandle, entryIdC, &totpCode)

        guard result == SPErrorCode_Success else {
            return nil
        }

        let code = String(cString: totpCode.code!)
        let seconds = totpCode.seconds_remaining

        sp_string_free(totpCode.code!)

        return TotpCode(code: code, secondsRemaining: seconds)
    }

    /// Generate password
    func generatePassword(length: Int, includeSymbols: Bool) -> String? {
        var passwordPointer: UnsafeMutablePointer<CChar>? = nil

        let result = sp_password_generate(
            UInt(length),
            includeSymbols ? 1 : 0,
            &passwordPointer
        )

        guard result == SPErrorCode_Success, let password = passwordPointer else {
            return nil
        }

        let passwordStr = String(cString: password)
        sp_string_free(password)

        return passwordStr
    }

    /// Check password strength
    func checkStrength(password: String) -> PasswordAnalysis? {
        let passwordC = password.cString(using: .utf8)!

        var analysis: SPPasswordAnalysis = SPPasswordAnalysis()

        let result = sp_password_check_strength(passwordC, &analysis)

        guard result == SPErrorCode_Success else {
            return nil
        }

        return PasswordAnalysis(
            score: Int(analysis.score),
            entropyBits: analysis.entropy_bits,
            crackTimeSeconds: analysis.crack_time_seconds,
            length: Int(analysis.length),
            hasLower: analysis.has_lower,
            hasUpper: analysis.has_upper,
            hasDigit: analysis.has_digit,
            hasSymbol: analysis.has_symbol
        )
    }

    /// Lock vault
    func lockVault() -> Bool {
        let result = sp_vault_lock(vaultHandle)
        return result == SPErrorCode_Success
    }

    /// Destroy vault when done
    func destroyVault() {
        _ = sp_vault_destroy(vaultHandle)
        vaultHandle = 0
    }
}

// Supporting types
struct EntrySummary {
    let id: String
    let title: String
    let username: String
    let favorite: Bool
}

struct TotpCode {
    let code: String
    let secondsRemaining: UInt32
}

struct PasswordAnalysis {
    let score: Int
    let entropyBits: Double
    let crackTimeSeconds: Double
    let length: Int
    let hasLower: Bool
    let hasUpper: Bool
    let hasDigit: Bool
    let hasSymbol: Bool
}
```

## Android Integration (Kotlin)

### 1. Add the Rust Library

In your `build.gradle`:

```gradle
android {
    // ...
    sourceSets {
        main {
            // Include sentinelpass-mobile-bridge JNI lib
            jniLibs.srcDirs += "path/to/sentinelpass-mobile-bridge/target/debug"
        }
    }
    ndk {
        abiFilters "arm64-v8a", "armeabi-v7a", "x86_64"
    }
}
```

### 2. Create Kotlin Wrapper

```kotlin
class SentinelPassVault(private val vaultPath: String) {

    private var nativeHandle: Long = 0

    init {
        System.loadLibrary("sentinelpass_mobile_bridge")
    }

    /**
     * Unlock or create the vault
     * Returns true on success
     */
    fun unlock(masterPassword: String): Boolean {
        nativeHandle = nativeInit(vaultPath, masterPassword)
        return nativeHandle != 0L
    }

    /**
     * Add a new password entry
     * Returns entry ID on success, null on error
     */
    fun addEntry(
        title: String,
        username: String,
        password: String,
        url: String = "",
        notes: String = ""
    ): String? {
        val result = nativeAddEntry(
            nativeHandle,
            title,
            username,
            password,
            url,
            notes
        )
        return result
    }

    /**
     * Get an entry by ID
     * Returns Entry as JSON string, null on error
     */
    fun getEntry(entryId: String): Entry? {
        val json = nativeGetEntry(nativeHandle, entryId) ?: return null

        // Parse JSON - you'll need to add a JSON library or implement parsing
        // For simplicity, using a basic approach:
        return parseEntryJson(json)
    }

    /**
     * List all entries
     * Returns list of EntrySummary, empty on error
     */
    fun listEntries(): List<EntrySummary> {
        val json = nativeListEntries(nativeHandle) ?: return emptyList()
        return parseEntryListJson(json)
    }

    /**
     * Generate TOTP code
     * Returns code string, null on error
     */
    fun generateTotp(entryId: String): String? {
        return nativeGenerateTotp(nativeHandle, entryId)
    }

    /**
     * Generate random password
     */
    fun generatePassword(length: Int, includeSymbols: Boolean): String? {
        return nativeGeneratePassword(nativeHandle, length, includeSymbols)
    }

    /**
     * Check password strength
     * Returns "score,description" format, null on error
     */
    fun checkStrength(password: String): String? {
        return nativeCheckStrength(nativeHandle, password)
    }

    /**
     * Lock the vault
     */
    fun lock() {
        nativeLock(nativeHandle)
    }

    /**
     * Clean up when done
     */
    fun destroy() {
        if (nativeHandle != 0L) {
            nativeDestroy(nativeHandle)
            nativeHandle = 0
        }
    }

    // Native method declarations
    private external fun nativeInit(
        vaultPath: String,
        masterPassword: String
    ): Long

    private external fun nativeDestroy(handle: Long)

    private external fun nativeIsUnlocked(handle: Long): Boolean

    private external fun nativeLock(handle: Long): Int

    private external fun nativeAddEntry(
        handle: Long,
        title: String,
        username: String,
        password: String,
        url: String,
        notes: String
    ): String?

    private external fun nativeGetEntry(
        handle: Long,
        entryId: String
    ): String?

    private external fun nativeListEntries(
        handle: Long
    ): String?

    private external fun nativeSearchEntries(
        handle: Long,
        query: String
    ): String?

    private external fun nativeDeleteEntry(
        handle: Long,
        entryId: String
    ): Int

    private external fun nativeGenerateTotp(
        handle: Long,
        entryId: String
    ): String?

    private external fun nativeGeneratePassword(
        handle: Long,
        length: Int,
        includeSymbols: Boolean
    ): String?

    private external fun nativeCheckStrength(
        handle: Long,
        password: String
    ): String?

    private external fun nativeBiometricHasKey(
        handle: Long
    ): Boolean

    private external fun nativeBiometricRemoveKey(
        handle: Long
    ): Int

    companion object {
        init {
            // Load native library
            System.loadLibrary("sentinelpass_mobile_bridge")
        }
    }
}

// Data classes for JSON parsing
data class Entry(

    val title: String,
    val username: String,
    val password: String,
    val url: String?,
    val notes: String?,
    val createdAt: String,
```

## Error Handling

### iOS Error Codes

```swift
enum SPErrorCode: Int {
    case success = 0
    case invalidParam = -1
    case vaultLocked = -2
    case notFound = -3
    case crypto = -4
    case database = -5
    case io = -6
    case alreadyUnlocked = -7
    case invalidPassword = -8
    case notInitialized = -9
    case biometric = -10
    case totp = -11
    case sync = -12
    case outOfMemory = -13
    case unknown = -99
}
```

### Android Error Codes

Same error codes are returned as `Int` values from native methods.

Check return values:
- `0` = Success
- Negative numbers = Error (see SPErrorCode enum above)

## Memory Management

### iOS
- All C strings returned **must be freed** with `sp_string_free()`
- Arrays **must be freed** with `sp_bytes_free(ptr, len)`
- Always call `sp_vault_destroy()` when done with a vault

### Android
- Strings are returned as jstring (managed by JVM)
- Complex types are returned as JSON strings
- The JVM manages memory automatically
- Call `destroy()` on your Vault wrapper when done

## Platform-Specific Notes

### iOS (Swift)
- Use `cString(using: .utf8)` for string conversion
- Cast C strings back with `String(cString:)`
- Be careful with pointer lifetimes
- Consider creating a Swift wrapper class

### Android (Kotlin)
- The package name is `com.sentinelpass` (configurable)
- JNI functions are prefixed with `native`
- Biometric key operations require platform keystore integration
- For byte arrays, you'll need to implement proper conversion

## Building

### iOS
```bash
# Build the mobile bridge for iOS (C ABI)
cargo build --package sentinelpass-mobile-bridge --release

# The static library will be at:
# target/release/libsentinelpass_mobile_bridge.a

# The C header is at:
# sentinelpass-mobile-bridge/include/sentinelpass_bridge.h
```

**Note:** A complete iOS SwiftUI app implementation is available at `ios/SentinelPass/`. See `ios/SentinelPass/README.md` for iOS-specific build instructions.

### Android
```bash
# Build the mobile bridge for Android (JNI)
cargo build --package sentinelpass-mobile-bridge --features jni

# The shared library will be at:
# target/debug/libsentinelpass_mobile_bridge.so
# (or libsentinelpass_mobile_bridge.a on older NDK)

# Copy to your Android project's jniLibs folder
```

To test the bridge integration:

1. Create a test vault with known credentials
2. Test vault init/unlock
3. Test adding/retrieving entries
4. Test TOTP generation
5. Test password generation
6. Test vault lock/unlock
7. Verify memory management (no leaks)

## Troubleshooting

### Common Issues

1. **"Library not loaded"** - Verify the native library is included in your app bundle
2. **"Unresolved symbols"** - Make sure you're building for the correct architecture (arm64, x86_64)
3. **Crash on unlock** - Check master password is correct and vault path exists
4. **Memory leaks** - Always free returned strings and arrays

### Debug Logging

The bridge does not include logging by default (security consideration). Add logging in your Swift/Kotlin wrapper to debug issues.

### Security Notes

- Never log master passwords or entry passwords
- Clear clipboard promptly after password operations
- Use platform keychains for biometric key storage
- The bridge does all crypto operations in Rust, not in Swift/Kotlin
